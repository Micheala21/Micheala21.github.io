### Reflection 
This reflection is about my journey of learning how to use Markdown and getting my GitHub Pages site up and running, including the challenges I faced along the way and what I learned from them. 

#### Reflection on Coding Markdown 
Learning Markdown was quite easy for me. Compared to other programming languages that can take weeks to learn, I was surprised at how quickly I picked it up. The main difficulty I had was getting used to the formatting rules, especially spacing and alignment. For example, headings like “# Micheala Martin” need to start right at the beginning of a line and cannot be centered, since Markdown follows strict syntax rules. I sometimes forgot this, which led to a few mistakes in my work. With more practice, I got better and made fewer errors. Overall, it taught me to be more careful and pay closer attention to detail when coding. 

#### Reflection on embedding my Mock Interview Video using HTML 
Embedding the video using Markdown was quite challenging for me. I searched YouTube for tutorials on how to do it, and I found different methods, but most of them didn’t work because my video file was too large. I then had to compress the video to make it smaller and upload it to my GitHub repository. After that, I tried embedding it using HTML, but nothing showed up in the README.md preview, which confused me at first. After doing more research, I realised I needed to first set up GitHub Pages for the video to display properly. Once I figured that out, everything started working. This helped me understand the process better and made me more confident when working with GitHub. 

#### Relection on using Github Pages 
Deploying GitHub Pages wasn’t difficult for me at first, because when I originally created my repository, I learned how to do it successfully. However, after I ran into issues with embedding my video, I completely forgot how the deployment process worked, which led to my mistake of the video not showing in my README document. I was a bit stuck at that point, so I went back and did some research and asked my classmate where I went wrong. With his guidance, I was able to understand my mistake and follow the correct steps again. In the end, I managed to deploy GitHub Pages successfully and fix the issue, which helped me feel more confident using it in future.
